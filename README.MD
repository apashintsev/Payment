Заготовка для Merchant based платежного сервиса для приёма криптовалют.

Мерчант регистрируется в сервисе и ему генерируются персональные криптосчета.
Приватные ключи от счетов лежат в сервисе в зашифрованном виде.
Мерчант подключает свой сервис, который должен принимать криптовалюту на счета пользователей.
Общение с сервисом происходит через HTTP.

Внешние методы:

v2/addresses/take - сгенерировать для пользователя персональный адрес для приёма криптовалюты
v2/withdrawal/crypto - вывод криптовалюты на указанный адрес
v2/invoices/create - создать инвойс


Авторизация сервиса мерчанта на платёжном сервисе(этот сервис) осуществляется посредством отправки двух заголовков:
X-Processing-Key – открытый ключ, который можно получить из учетной записи мерчанта.
X-Processing-Signature – тело POST, подписанное секретным ключом HMAC-SHA512, секретный ключ также получается из учетной записи мерчанта.

После успешной транзакции сервис отправляет коллбэк на указанный мерчантом адрес, так же зашифрованный и подписанный публичным ключом
